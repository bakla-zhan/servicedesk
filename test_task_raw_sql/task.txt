целью проверочного задания является проверка понимания и навыков:
1. организации REST сервисов
2. процедуры аутентификации
3. кодогенерации
4. работы с бд
5. работы с внешним сервисом
6. написания unit тестов
7. минимальных навыков рвзвертывания сервиса

данное задание по сути не отражает реальные задачи работодателя, но проверяемые навыки соискателя будут необходимы в работе

необходимо написать систему регистрации обращений
1. REST сервис, генерируемый на основе swagger спецификации
2. аутентификация по API-ключу в заголовке запроса (ключ можно указать просто переменной, т.к. не требуется организация авторизации)
3. операции добавить/посмотреть/обновить/получить список/удалить обращение
4. обращение содержит поля: заголовок, тело обращения, email автора
5. сохранять обращения в бд postgres (заюзать GORM)
6. осуществлять проверку email автора, используя внешний сервис https://dadata.ru/api/clean/email/ (надо зарегаться и можно бесплатно ограниченное количество запросов юзать) - нужно отклонять запросы с типом проверки 3, т.е. с одноразовых адресов
7. написать unit тесты (на работу с внешним сервисом точно, на остальной код - если есть время/необходимость)
8. описать Dockerfile непосредственно для сервиса и базы и написать конфиг docker-compose
9. необходимые параметры для работы сервиса указать переменными окружения и/или параметрами командной строки при запуске приложения
